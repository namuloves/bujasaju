# thegot.co.kr 사주 풀이 LLM 역공학

**캡처 방법:** WordPress 서버 사이드 렌더링 — `/saju-result?y=&mo=&d=&g=&h=&mn=...` 쿼리스트링 GET 한 방으로 끝. Chrome 확장 필요 없음. `curl` + Python `re` 파싱으로 prose 본문 그대로 빼옴.

**캡처한 두 샘플:**
- 샘플 1 — 1990-01-01 12:00 양력 남성 → [thegot-r1-1990-01-01-male.txt](./thegot-r1-1990-01-01-male.txt) (51문단, 11섹션)
- 샘플 2 — 1985-06-15 03:30 양력 여성 → [thegot-r2-1985-06-15-female.txt](./thegot-r2-1985-06-15-female.txt) (48문단, 11섹션)

## 핵심 발견 — thegot은 "**컴퓨테드 사주 데이터를 LLM에 통째로 박아넣는다**"

두 샘플 비교했을 때 명백한 패턴:

### 1. 섹션 구조는 100% 고정 템플릿

11개 섹션 순서 동일:
1. 가장 ___한 부분부터 (인사+격국 헤드)
2. 나는 어떤 사주인가 (일주+격국+용신)
3. 지금 이 시기의 의미 (현재 대운)
4. 직업운
5. 재물운
6. 연애운
7. 건강운
8. 조상·부모·자녀궁 분석
9. 올해 운세와 월별 흐름
10. 다음 대운
11. 마지막으로 드리는 말 : 개운법

**섹션 제목조차 일부 변주 가능** (1번이 "가장 중요한 이야기부터" vs "가장 궁금하실 부분부터", 11번이 "마무리 : 당신만의 개운법" vs "마지막으로 드리는 말 : 개운법") — LLM이 **제목까지 살짝 다시 쓰게** 함. 자연스러움 만드는 작은 디테일.

### 2. 사주 계산은 결정적, 풀이는 LLM

**결정적 (코드로 미리 계산):**
- 일주 (병인 / 을유)
- 격국 (정관격 / 식신격)
- 용신 (수 / 목)
- 강약 판정 (신강 / 중화)
- 오행 분포 카운트 (각 5개 원소 개수)
- 십성 분포 카운트
- 합·충·형·파·해 목록 (갑기합토, 인오술 화국, 자오충, 무은지형, 인사해, 공망)
- 신살 트리거 (도화살, 양인살, 화개살, 현침살 등)
- 대운 사이클 (32세 경진 / 42세 신사 — 정확한 사이클 + 십성)
- 세운 (2026 병오)
- 월운 12개 (병오년 1월~12월 각각 천간지지+십성)

**LLM이 쓴 부분 (풀이 prose):**
- 모든 섹션의 본문
- 비유 ("봄 숲에서 태양이 힘차게 떠오르는 형상", "가을 서리를 맞은 꽃")
- 구체적 행동 권고 ("벌자마자 30%는 무조건 따로 빼놓는 습관")
- 월별 추천 ("11월(기해) — 상관의 기운이 들어와서 창의력이 폭발")
- 종합 클로징

### 3. 풀이 안에 **숫자 fact**가 빽빽하게 박혀있다

이게 thegot 풀이가 generic하지 않은 핵심 이유. 모든 권고 문장에 숫자/이름/날짜가 강제로 박혀있어요:

샘플 1에서 추출한 fact 인스턴스:
- "**42세부터 신사 대운**이 시작되면 흐름이 바뀌어요" — 정확한 시작 나이
- "**52~61세(임오 대운), 62~71세(계미 대운)**" — 미래 사이클들
- "**11월(기해)**", "**12월(경자)**" — 좋은 달 이름까지 정확히
- "**6월(갑오)**", "**8월(병신)**", "**9월(정유)**" — 조심할 달
- "**관성(수)이 1개**", "**재성(금)이 원국에 없어요**" — 십성 카운트
- "**2026 병오년**", "**경진 대운**" — 정확한 60갑자 라벨

이건 **LLM이 만든 게 아니에요**. 코드가 미리 계산해서 LLM 프롬프트에 컨텍스트로 박아준 거예요. LLM은 그 fact를 prose로 끼워넣기만 함.

### 4. 같은 표현이 두 샘플에서 그대로 반복됨 → 템플릿 증거

LLM이 진짜 free-form이면 표현이 매번 달라야 해요. 그런데:

**100% 동일 문장:**
- 샘플 1 §7: "사주 오행으로 건강 취약점을 살펴볼게요."
- 샘플 2 §7: "사주 오행으로 건강 취약점을 살펴볼게요." ← 토씨 하나 안 틀림

**거의 동일 (장기 매핑 + 음식 매핑이 미리 짜인 슬롯):**
- 샘플 1 §7: "**심장, 소장, 혀, 혈액** 쪽은 화 기운이 과한 편이라 고혈압, 심장 두근거림, 불면증, 조급함 등의 증상이 나타날 수 있어요. 쓴맛 음식(녹차, 쑥)이 도움이 되고, 명상이나 심호흡으로 마음을 안정시키는 게 좋아요."
- 샘플 2 §7: "**심장, 소장, 혀, 혈액** 쪽은 화 기운이 부족해서 관련 기능이 약간 부족할 수 있어요. 쓴맛 음식(녹차, 쑥)이 도움이 되고, 명상이나 심호흡으로 마음을 안정시키는 게 좋아요." ← 화 기운 대상이 두 사주 다 화이고, 후반부 동일

**완전 동일 → 코드가 직접 출력한 템플릿:**
- 샘플 1: "재물 쪽은 솔직히 힘든 부분이 있어요. 타고난 재물 구조가 약하거든요. 하지만 대운의 흐름을 타면 조금씩 나아지는 시기가 옵니다. 과한 욕심보다 꾸준함이 이 사주의 재물 전략이에요."
- 샘플 2: 위 문단 토씨 하나 안 틀리고 동일하게 등장

LLM 출력이면 토씨 하나도 같을 수 없음. 이건 **PHP/JS 템플릿이 직접 박아넣은 문장**.

### 결론 — thegot은 **하이브리드 시스템**

- **70-80%는 코드 + 미리 쓴 템플릿** (조건부 분기로 채워지는 빈칸 채우기 문장)
- **20-30%는 LLM** — 섹션 도입부, 일주 비유, 한 줄 정의, 종합 클로징

LLM이 "전체 prose 생성기"가 아니라 "**템플릿 사이를 부드럽게 잇는 접착제**"로 쓰임. generic하지 않은 비결.

## 추정 시스템 아키텍처

```
[입력: 생년월일시 + 성별]
        ↓
[1. 만세력 계산 — 결정적]
   → 사주 4기둥 (천간/지지)
   → 일간/일주/월지/격국/용신
   → 오행 분포 / 십성 분포
   → 합·충·형·파·해 라벨
   → 신살 트리거 리스트
   → 대운 사이클 8-10개
   → 세운 + 월운
        ↓
[2. 룰 기반 큰 결정]
   → 신강/중화/신약 판정
   → 길운/흉운 판정 per 대운
   → 좋은 달 / 조심할 달 (월운 십성 → 사전 매핑)
   → 추천 색/방향/음식 (용신 → 사전 매핑)
        ↓
[3. 템플릿 빈칸 채우기 + LLM 호출 혼합]
   → 단순 fact 문장: 코드가 직접 (재성 카운트 등)
   → 비유·도입·전환·클로징: LLM이 한 호출에 다 처리
   → 입력으로:
     • 11섹션 골격 (제목 + 각 섹션이 다뤄야 할 fact)
     • 페르소나 / 톤 가이드
     • 좋은 예 / 나쁜 예 (few-shot)
        ↓
[4. 후처리]
   → 강조 단어 <b>로 감싸기 (격국명, 한자, 신살명)
   → 내부 링크 자동 삽입 (대운 → /major-luck-cycle/)
   → 광고 슬롯 끼워넣기
   → HTML 렌더 (sj-consult-section / sj-consult-p)
```

## 추정 LLM 모델 / 호출 방식

WordPress 서버 사이드 렌더라서 네트워크 탭 안 봐도 됨 — 결과 페이지가 한 번에 떨어진다는 사실 자체가 단서.

**모델 후보:**
- 가장 가능성 높음: **GPT-4o** 또는 **GPT-4o-mini** — 한국어 이런 톤 자연스럽고 한자 인용 안정적
- 가능성 있음: **Claude Haiku 3.5 / Sonnet** — 같은 톤 가능
- 덜 가능성: **Solar / HyperCLOVA X** — 비유 디테일이 영문 모델 톤에 가까움

**한 번 호출 vs 11번 호출:**
- 11섹션 prose 평균 4-5문단, 총 약 4000자. gpt-4o-mini 한 번 호출로 충분.
- 페이지 응답이 1-2초 안에 떨어지는 걸 봐서 streaming 안 함, 한 번에 받음.
- 또는 §1-3 / §4-7 / §8-11 식 3분할 병렬도 가능.

**추정 system prompt 구조:**

```
당신은 명리학 30년 경력의 상담사입니다. 격식은 차리되 친근한 해요체로,
사주 한 사람을 30분간 마주앉아 상담하듯 풀어주세요.

[규칙]
1. 절대 모르는 정보는 만들지 마세요. 아래 [사주 분석] 섹션의 fact만 사용하세요.
2. 모든 권고에는 구체적 숫자/이름/시기를 박으세요. 일반론 금지.
3. 한자 용어는 첫 등장에 (한자) 병기. 두 번째부터는 한글만.
4. 한 문단은 2-4문장. 너무 길어지면 끊으세요.
5. 11섹션 순서대로. 섹션 제목은 가이드에서 살짝 변주 가능.
6. 강조하고 싶은 격국명/신살명/숫자는 **굵게** 처리.
7. "넘치는 힘에 출구가 없는 사주" 같은 한 줄 정의를 1번 섹션에 넣으세요.

[페르소나 톤 예시]
- "사주를 보니 기운이 참 단단한 분이시네요. 풀어드릴게요."
- "솔직히 말씀드리면, 지금은 좀 버거운 시기예요."
- "벌자마자 30%는 무조건 따로 빼놓는 습관이 필요합니다."
```

**추정 user prompt (각 사주마다 다름):**

```

[십성 분포]
비견 1, 겁재 0, 식신 0, 상관 1, 편재 0, 정재 0,

일지 십성: 편인

[합충형파해]
천간합: 갑기합토
삼합/반합: 인오술 화국 (반합)
충: 자오충 / 형: 무은지형 / 해: 인사해
공망: 술·해

[신살] 도화살(일지), 양인살(월지), 역마살(시지), ...

[대운 사이클]
2~11세: 정축 (편인 정인) — 길운
12~21세: 무인 (식신 편인) — 평
22~31세: 기묘 (상관 편인) — 평
32~41세: 경진 (편재 식신) — 흉운 ← 현재
42~51세: 신사 (정재 비견) — 길운
52~61세: 임오 (편관 겁재) — 평
62~71세: 계미 (정관 식신) — 길운

[세운]
2026 병오년 — 비견 겁재 (기신)
좋은 달: 11월(기해, 상관), 12월(경자, 편재)
조심할 달: 6월(갑오, 편인), 8월(병신, 비견), 9월(정유, 겁재)

[개운법 매핑]
색: 검정/남색 / 방향: 북 / 음식: 짠맛, 해산물

위 사주 데이터로 11섹션 풀이를 작성해주세요.
1. 가장 중요한 이야기부터 (격국 핵심 + 한 줄 정의 + 현 대운 평가)
2. 타고난 기질과 구조 (일주 형상 비유 + 격국 + 용신 위치)
3. 이 시기를 어떻게 보낼 것인가 (현 대운 + 세운 + 27-71세 흐름 한 줄)
4. 연애운 (관성 카운트 → 표현 / 미래 관성 대운 / 배우자 오행)
5. 직업운 (격국 → 직군 / 신살 → 강점 / 이직 타이밍)
6. 재물운 (재성 카운트 → 톤 / 식상생재 등 구조 / 전략)
7. 건강운 (일간 오행 → 장기 / 과다·부재 오행 → 증상+음식 / 좋은 달·나쁜 달)
8. 조상·부모·자녀궁 (년주 십성 / 월주 십성 / 시주 십성 — 각 1문단)
9. 올해, 이렇게 보내세요 (세운 + 좋은 달 + 조심할 달 + 마무리)
10. 앞으로의 흐름 (다음 대운 1줄)
11. 마무리 : 개운법 (신살 강조 + 색/방향/음식 / 한 줄 클로징 키워드 6개)
```

이런 구조면 LLM은 생성보다 **포맷팅**에 가까운 작업을 하게 됨. 그래서 fact가 안 빠지고, 일관된 톤이 유지되고, 일반론이 안 나옴.

## bujasaju 현재 vs thegot

[src/app/api/saju-summary/route.ts](../src/app/api/saju-summary/route.ts) 현재 (gpt-4o-mini, streaming):

| 차원 | bujasaju 현재 | thegot |
|---|---|---|
| 사전 계산 fact | 일주, 월지, 격국, 일간 + 충/합/형 분석 + ohaeng 컨텍스트 | 위 모든 것 + 신살 + 대운 사이클 + 세운 + 월운 + 좋은달/나쁜달 + 개운법 매핑 |
| 출력 길이 | 5-6문장 | 약 50문단 / 4000자 |
| 섹션 구조 | 1개 (단일 문단) | 11개 |
| 페르소나 | "40년 경력 명리학 대가" | 추정 "30년 경력 상담사" |
| 톤 | 해요체 강제 | 해요체 강제 |
| 금지 리스트 | 매우 구체적 | 추정 비슷 + "11섹션 절대 준수" 추가 |
| 숫자/이름 강제 | 없음 | 모든 권고에 숫자/달/나이 박힘 |
| LLM 역할 | 모든 prose 생성 | 미리 계산된 템플릿 사이 접착제 |

**가장 큰 차이:** bujasaju는 LLM에게 **너무 많은 자유**를 줌. 5-6문장 안에 일주/월지/격국/오행/매칭부자 다 녹이라고 하니 결국 추상적 일반론만 나오는 거. thegot은 LLM이 **방향만 잡고 fact는 코드에서 박아주는** 구조.

## bujasaju에 적용할 구체적 변경 3개

### 변경 1 — 섹션 분할 강제

현재: "5-6문장 하나의 글로". 한국어 LLM은 이 길이 안에서 깊이 못 만듦.
바꿀 것: **섹션 7-11개로 명시 분할**. 각 섹션 2-4문장. 총 14-30문장.

### 변경 2 — 미리 계산된 숫자/이름 fact를 prompt에 박기

현재: `buildSajuContext()`가 prose 형태 컨텍스트만 줌.
바꿀 것: 다음을 추가로 prompt에 박아 넣음:
- 오행 분포 raw counts (목 N, 화 N, ...)
- 십성 분포 raw counts
- 신살 list (트리거된 것만, 어느 주에 있는지)
- 합/충/형/파/해 라벨 list
- 대운 현재 + 다음 사이클 (몇 살 ~ 몇 살, 십성, 길흉)
- 세운 2026 + 좋은 달 / 조심할 달

그리고 prompt에 "**모든 권고 문장에는 위 데이터의 구체 숫자/달/나이를 최소 1개 박으세요**" 강제.

### 변경 3 — few-shot 예시 추가

현재: 좋은 예/나쁜 예가 1쌍씩만.
바꿀 것: thegot 출력에서 뽑은 "좋은 문단 5개 + 나쁜 문단 5개" 예시. 특히 thegot의 "이 사주를 한마디로 정의하면, '넘치는 힘에 출구가 없는 사주'예요" 같은 한 줄 정의 패턴은 LLM이 따라하기 좋은 템플릿.

## 캡처 cheat code (재현 가능)

원하는 사주 풀이를 그냥 받고 싶으면:

```
curl -L "https://thegot.co.kr/saju-result?y=YYYY&mo=M&d=D&g=M|F&h=H&mn=MN&yj=0&lc=0&st=1" \
  -A "Mozilla/5.0" -o page.html
```

음력이면 `&cal=lunar&leap=0`. 결과 HTML 안에 `class="sj-consult-p"` 노드들이 풀이 본문. 11개 sj-consult-section. WordPress server-side render라서 클라이언트 / 헤드리스 브라우저 / Chrome 확장 다 필요 없음.
