# thegot 사주 fact 인벤토리 — 우리가 계산해야 할 것 vs 이미 가진 것

raw extraction: [thegot-r1-facts-raw.txt](./thegot-r1-facts-raw.txt) (1990-01-01 12:00 남자 기준)

thegot이 LLM 호출 전에 코드로 **미리 계산해놓은** fact를 카드별로 다 까봤어요. 우리한테 이미 있는 것 / 코드 일부만 있는 것 / 아예 없는 것 표시.

---

## 카드 1 — 사주팔자 원국 (4기둥)

**thegot 출력:**
```
시주 갑오  (편인 갑목 / 겁재 오화)
일주 병인  (병화 / 편인 인목)
월주 병자  (비견 병화 / 정관 자수)
년주 기사  (상관 기토 / 비견 사화)
```
+ 각 자리에 십성 라벨 + "어린 시절의 환경" / "직장·사회 환경" / "노년의 환경" 같은 **궁위 의미** 라벨.

**우리:** ✅ 4기둥 + 십성 — `tenGods.getSipSin()` 있음.
**갭:** 궁위 라벨링 ("년간=조상", "월간=사회에서 보이는 모습", "시지=노년의 환경·자녀 인연" 등) — **8문장짜리 정적 사전 1개 만들면 끝.**

## 카드 2 — 일주 해석

**thegot 출력:**
- 일주 별명 ("새벽 해" — 60갑자별 별명)
- 성격/직업/연애/건강/행운 키워드 5개씩
- **같은 일주 유명인 4명** (스티븐 스필버그, 넬슨 만델라, 간디, 제이크 질렌할)
- **이성 궁합 찰떡 3개 / 상극 3개** (60갑자 쌍 매핑)
- **동성 궁합 찰떡 3개 / 상극 3개**
- **오늘의 일운** (오늘 날짜의 일진 무인 + 일간과의 십성 + 점수 + 영역별 한 줄)

**우리:**
- ✅ 60갑자 일주 풀이: `ilju.json` (60개 풍부)
- ❌ 60갑자 별명 ("새벽 해", "바다 위 거목" 등) — 없음
- ✅ 같은 일주 유명인 — 우리는 부자 매칭으로 더 강함, 이건 우리가 thegot보다 위
- ❌ 이성/동성 궁합 찰떡·상극 매트릭스 (60×60)
- ❌ 오늘의 일운 (현재 시각 일진 lookup)

**갭:** 별명 60줄 (콘텐츠), 궁합 매트릭스 (60×60 = 3600칸이지만 일주 5요소 기준으로 룰 기반 계산 가능 — 코드 ~50줄), 일진 lookup (`solar-terms-jie.json`에 절기 있으니 일진은 평행한 데이터 한 번 더 필요).

## 카드 3 — 오행 분포

**thegot 출력:**
- 5원소 카운트 + % (목 2/25%, 화 4/50%, 토 1/13%, 금 0/0%, 수 1/13%)
- 십성 카테고리 라벨 (비겁/식상/재성/관성/인성)
- **일간 기준 오행 관계** (식상/인성/재성/관성 누가 몇 개)
- **균형 진단** ("약간의 편중", "큰 문제 없어요")
- **부족·과다 오행마다 한 줄 + 보완책** ("금 부족 → 흰색/금색, 서쪽 방향")

**우리:**
- ✅ 5원소 카운트 — `sajuContext.ts countOhaeng()` 있음
- ❌ % 변환 + 십성 카테고리화 — trivial 추가
- ❌ "약간 편중" / "큰 문제 없음" / "심각한 결핍" 같은 **단계 라벨링 임계값**
- ✅ 부족/과다 보완책 prose — `ohaeng-analysis.json whenAbsent`/`whenExcess` 있음
- ❌ 색/방향/숫자 매핑 사전 (오행 → 색·방향·숫자·계절·음식·직업·소재·운동) — 정적 사전 5×8칸

## 카드 4 — 사주 심화 풀이 (★ 핵심 카드 ★)

이게 thegot의 진짜 무기. 풀이의 80%는 이 카드의 fact에서 나옴.

**thegot 출력:**
- **강약 점수 (53점, 신강)** — 0~100 스케일
- **득령/득지/득시/득세 4지표** — 월령/일지/시지/사주 전체에서 일간이 힘 받는지 boolean × 4
- **3개 학파 용신 동시 표시:**
  - 격국용신 = 목
  - 조후용신 = 수
  - 억부용신 = 토
  - ★ 표시로 "우선 풀이 기준" (학파 다툼 정치적으로 회피하는 영리한 트릭)
- 용신·희신·한신·기신·구신 5단계 분류
- **십성 조합 패턴 라벨링:**
  - 관인상생 (官印相生)
  - 상관견관 (傷官見官)
  - 비겁쟁재 (比劫爭財)
  - 식상생재, 살인상생 등 ~15-20개 패턴
- 각 패턴마다 1-2문장 진단 + 조언

**우리:**
- ❌ 강약 점수 — 없음. **이게 가장 큰 갭.** 점수화 알고리즘은 (월령 가중치 + 일지·시지 + 인성·비겁 카운트 — 식상·재성·관성 카운트) 류로 짜면 됨. ~80줄.
- ❌ 득령·득지·득시·득세 — 강약 점수 부산물. ~20줄.
- ⚠️ 용신 — `ohaeng-analysis.json dayMasterNeeds` 있음 (격국용신 풍의 매핑) but 학파 3종 동시 X
- ❌ **십성 조합 패턴 라벨링** — 우리한테 가장 인상적이었던 부분. "관인상생", "상관견관" 같은 라벨이 풀이 prose에 그대로 박힘 ("가장 큰 화두는 바로 상관견관 구조에 있어요"). 패턴 ~20개 룰 + 라벨 + 진단. ~150줄 코드 + 콘텐츠.

## 카드 5 — 운세 흐름 (대운 + 세운 + 월운)

**thegot 출력:**
- **대운 시작 나이 (2세) + 시작 날짜 (1991-07-10)** — 정확한 절기 기반 계산
- 대운 8칸 (2-11세 정축 / 12-21세 무인 / ... / 72-81세 갑신) — 천간·지지·오행·십성 모두 라벨링
- 색깔로 길흉 표시 (파랑=길, 무색=평, 빨강=흉) — **용신 매칭으로 자동 판정**
- **연운 1991~2070 (80년치)** — 매년 천간/지지/십성/12운성 계산
- **2026년 월운** — 12달 × (천간/지지/십성 × 2)
- **현재 대운 상세 풀이** — 32~41세 경진 = 편재 대운 → "사업, 투자, 이성" 키워드 + 4-5개 조언
- **올해 세운 한 줄 + 이번 달 월운 한 줄 + 오늘 일운 한 줄**

**우리:**
- ✅ 대운 계산기 있음 — `daewoon.ts calculateDaeUn()` (양/음남녀 순역행, 절기 기반 시작 나이, 월주 ±순환)
- ✅ 대운 십성: `getDaeUnSipSin()`
- ❌ **길흉 색 자동 판정 (대운별)** — 용신/기신 매핑만 짜면 trivial
- ❌ 연운 80년치 자동 계산 — 60갑자 순환이라 trivial 한 30줄
- ❌ 월운 12달 자동 — 60갑자 순환 + 월간 공식
- ❌ 십성별 대운 풀이 — `daewoon.json 십성별대운` 있음 but 미사용

**갭:** 코드 200줄 정도 + 길흉 자동 판정 룰. 대운 코어는 이미 있음.

## 카드 6 — 사주 관계 (합·충·형·파·해·공망)

**thegot 출력:**
- 모든 관계 자동 라벨링 (천간합 갑기합토 / 반합 인오술 화국 / 자오충 / 무은지형 / 인사해 / 술해 공망)
- 각 관계마다 **어느 주 ↔ 어느 주에 있는지** 명시 ("월지↔시지", "년간↔시간")
- 관계 종류별 부연 설명 (3-4문장)

**우리:**
- ✅ 합/충/형 다 있음 — `relationships.ts` (천간합 5, 육합 6, 삼합 4, 방합 4, 충 6, 형 4그룹)
- ⚠️ 파/해/원진 — `hapchung.json` 데이터는 있지만 evaluator 코드에 있는지 확인 필요
- ❌ 공망 (空亡) 계산 — 일주 60갑자별 갑자순 매핑으로 결정. ~30줄 데이터 + 5줄 로직.
- ❌ 위치 라벨 ("월지↔시지") 자동 부착 — 한 라인.

## 카드 7 — 신살 (★ 14개 동시 트리거 ★)

**thegot 출력:** 한 사주에서 **14개** 신살이 트리거됨. 카테고리 라벨링 (귀인성 4 / 매력성 2 / 권력성 1 / 영성 1 / 위험성 3 / 기타 2):
- 학당귀인, 태극귀인, 복성귀인, 정록 (귀인성)
- 도화살, 홍염살 (매력성)
- 양인살 (권력성)
- 천문성 (영성)
- 겁살, 망신살, 육해살 (위험성)
- 연살, 현침살×2 (기타)

각 신살마다: 한자 + 어느 주에 있는지 + 1-2줄 풀이 + 💡 조언.

**우리:**
- ⚠️ `sinsal.json`에 8개 신살 데이터 있음. **thegot이 쓰는 14개 중 5-6개만 우리한테 있음.**
- ❌ 신살 evaluator (rule → boolean 판정 코드) 없음
- ❌ 카테고리 라벨링 (귀인성 / 매력성 / 권력성 / 영성 / 위험성)

**없는 신살 (thegot은 갖고 우리는 없음):** 학당귀인, 태극귀인, 복성귀인, 정록, 홍염살, 천문성, 망신살, 육해살, 연살, 현침살.

**갭:** 신살 evaluator 코드 ~150줄 + 새 신살 10개 콘텐츠 + 카테고리 매핑.

## 카드 8 — 12운성

**thegot 출력:**
- 4기둥 각각의 12운성 라벨 (시주=제왕 / 일주=장생 / 월주=태 / 년주=건록)
- **에너지 점수 (12/12, 9/12, 2/12, 11/12)** — 12운성 단계별 강도 매핑
- 각 주별 1-2줄 풀이

**우리:**
- ✅ 12운성 매핑 테이블 — `sajuContext.ts TWELVE_STAGE_TABLE` (이미 있음, 일지에만 적용 중)
- ✅ 12운성 풀이 prose — `twelve-stages.json` 있음
- ❌ 에너지 점수 매핑 (12개 단계 → 1-12 강도) — 12줄 정적 사전
- ❌ 4기둥 모두에 적용 (현재 일지만) — for 루프 한 번

## 카드 9 — 납음오행

**thegot 출력:**
- 4기둥 각각의 납음오행 (년주 대림목 / 월주 간하수 / 일주 노중화 / 시주 사중금)
- 일주 납음 상세 풀이 ("화로 속의 불 — 안정적이고 지속적인 열")
- 성격/직업/조언 한 줄씩

**우리:**
- ❌ 납음오행 데이터/계산 자체가 없음.
- 60갑자 → 30개 납음 매핑 (정적 표) + 30개 납음별 풀이 prose.

**갭:** 60줄 매핑 사전 + 30개 풀이 (각 3문장) ≈ 100줄 콘텐츠.

## 카드 10 — 공망

**thegot 출력:**
- 일주 갑자순 → 공망 두 지지 (술·해)
- 사주 안에 공망 지지가 있는지 boolean

**우리:**
- ❌ 공망 계산 자체 없음.

# 진짜 핵심 두 개

다른 거 다 미루더라도 **이 두 개만 짜면** thegot 풀이 깊이의 80% 달성:

### 1. 강약 점수 + 3종 학파 용신 (~200줄)

thegot 풀이에서 모든 권고의 근거가 이 두 값. "신강이라 출구가 필요하다", "용신 수가 부족해서 11월·12월에 들어올 때 움직이세요" — 강약과 용신이 정해지면 나머지 모든 prose가 따라옴.

### 2. 십성 조합 패턴 라벨링 (~200줄)

thegot 풀이의 "**상관견관 구조**", "**관인상생**", "**비겁쟁재**" 같은 라벨은 이 카드의 자동 패턴 매칭 결과. 4글자 한자 라벨 하나만 prose에 박혀도 풀이가 갑자기 깊어 보임. 약 15-20개 패턴 룰이 표준.

이 두 개만 짜놓으면 LLM prompt에 박을 fact가 비약적으로 풍부해짐. 이게 가장 큰 ROI.
