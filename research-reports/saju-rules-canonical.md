# 사주 풀이 규칙집 — 1차 자료 기준 정리

**목적:** 우리 사이트의 사주 해석을 정확하게 만들기 위해, 표준 명리학 룰을 가능한 1차 자료(자평진전·궁통보감·연해자평·적천수) 기준으로 정리. 우리가 이미 아는 부분, 모르는 부분을 명시하고, 모르는 부분은 어떤 학파/책 기준을 따를지 결정.

**중요:** 출처가 다른 자료끼리 충돌이 있는 경우(예: 도화살 트리거를 잘못 적은 사이트도 있음), 책 1차 출처 우선. 이 문서가 우리 사이트의 "공식 입장" 역할.

| 영역 | 기준 책 | 한국어 표준 번역본 |
|---|---|---|

## 3. 십성(十神) 라벨링

**룰:** 일간 기준으로 다른 천간(또는 지장간)과의 오행 + 음양 관계로 10라벨.

| 오행 관계 | 음양 같음 | 음양 다름 |

## 4. 강약 판정

### 4.1 4지표 (boolean)

자평진전 + 적천수 표준 정의:

**득령(得令):** 월지 본기가 일간 입장에서 **인성(印) 또는 비겁(比劫)**이면 득령.
**득지(得地):** 일지 본기가 인성 또는 비겁이면 득지.
**득시(得時):** 시지 본기가 인성 또는 비겁이면 득시.
**득세(得勢):** 일간 자신을 제외하고 사주 8글자 중 **인성 + 비겁이 3개 이상**이면 득세.

### 4.2 등급화 (점수 X)

자평진전·적천수에 점수 개념 없음. 4지표 boolean 조합으로 분류:

| 등급 | 조건 |
|---|---|
| 최강 | 득령 + 득지 + 득세 |
| 중강 | 득령 + 득세, 실지 |
| 강 | 득령 + 득지, 실세 |
| 약변강 | 실령 but 득지 + 득세 |
| 중화 | 1-2 지표 충족, 균형 |
| 강변약 | 득령 but 실지 + 실세 |
| 중약 | 1지표만 충족 |
| 약 | 실령 + 실지, 득세 |
| 태약 | 실령 + 실지 + 실세, 인성·비겁 거의 없음 |
| 극왕 | 비겁이 5개 이상 → 종강격 후보 |
| 극약 | 비겁·인성 0개 → 종약격 후보 |

**우리 결정:** 점수 표시 안 함. 4지표 + 등급만 표시.
**우리 상태:** ❌ 강약 판정 함수 없음. 새로 작성.

### 4.3 왜 점수를 안 쓰는가

- 자평진전·적천수에 점수 산식 없음 — 현대 사이트가 임의로 만든 것
- 사이트마다 가중치 다름 (검증 불가능한 수치)
- "53점" 같은 숫자가 사용자한테 의미 없음 (어차피 등급으로 변환)
- 등급만 정확하면 후속 단계(용신 결정 등)에 충분

## 5. 격국 분류

### 5.1 결정 절차 (자평진전)

원칙: **"八字用神，專求月令"** — 격은 월령에서 구한다.

```
1. 월지의 지장간을 펼친다 (본기/중기/여기).
2. 본기가 천간 4개(년·월·일·시간) 중 어디에 투간했는지 확인.
   → 투간했으면 그 본기에 해당하는 십성 = 격.
3. 본기 미투간이면 중기 투간 확인 → 중기 십성 = 격.
4. 중기도 미투간이면 여기 투간 확인 → 여기 십성 = 격.
5. 셋 다 투간 안 했으면 → 월지 본기의 십성을 그대로 격으로 채택.
```

**예외:** 월지가 일간의 록(祿)지면 → 건록격. 양간 일간의 양인지면 → 양인격.

### 5.2 8격 + 2 = 10정격

**4길신(순용):** 정관격 / 정재격 / 정인격 / 식신격 — 격을 보호·생조하는 방향이 좋음.
**4흉신(역용):** 편관(칠살)격 / 상관격 / 편인격 / 겁재격 — 격을 제어·억제하는 방향이 좋음.
**+ 건록격 / 양인격** = 총 10정격(또는 12정격으로 비견격·편재격 추가).

우리 `gyeokguk.json`에 12개 있음.

### 5.3 변격(외격)

사주가 한 오행에 극단적으로 치우쳤을 때 정격 룰을 따르지 않고 흐름에 종속:
- **종강격(從强格):** 비겁 압도적 (5+개)
- **종왕격(從旺格):** 비겁 + 인성 압도적
- **종재격(從財格):** 일간 약 + 재성 5+
- **종관살격(從官煞格):** 일간 약 + 관성 5+
- **종아격(從兒格):** 일간 약 + 식상 5+
- **화격(化格):** 일간이 다른 천간과 합화
- **양기성상격:** 사주가 두 오행만으로 구성

**임계값 결정:** 자평진전에 명확한 숫자 없음. 한국 명리학계 다수 의견은 **80%** (천궁명리·플러스만세력 등 다수 사이트 합의):
- 종강격: 비겁+인성 80% 이상 + 관살·재 거의 없음
- 종왕격: 비겁만 80% 이상
- 종재격: 재성 80% + 인성·비겁 없음
- 종관격: 관살 80% + 인성·비겁 없음
- 종아격: 식상 80% + 인성·비겁 없음

**우리 사이트 채택값:** 80% = 사주 8글자 중 **7개 이상**이 한 카테고리. 책에 명시값 없으므로 사이트 명시적 결정으로 푸터 표기.

**우리 상태:** ✅ 정격 12개 데이터 있음, 결정 함수 일부(`gyeokguk.ts`).
**할 일:** 본기→중기→여기 투간 절차, 변격 fallback.

## 6. 용신 결정

### 6.1 학파 3종

#### 억부용신(抑扶用神) — 강약 기반 (적천수 우선)

```
신강 → 일간 빼내기:
  • 식상 (배출)
  • 재성 (소모)
  • 관성 (제압)
  세 후보 중 사주에 있고 힘 있는 것 우선.

신약 → 일간 보강:
  • 인성 (생함)
  • 비겁 (같은 편)
  둘 중 적합한 것.

중화 → 미세 조정. 약간 기운 쪽의 반대.
```

전체 사주의 약 70%가 이 방식.

**대안 — 4분류 룰 기반 임시 채택:**

1. **한기 다급** (해·자·축월) → 화(火) 용신
2. **열기 다급** (사·오·미월) → 수(水) 용신
3. **조기(마름)** (미·술 + 화 다수) → 수(水) 보강
4. **습기(축축함)** (진·축 + 수 다수) → 화(火) 보강

이건 80% 정확도. 책 표 확보 시 100%.

**우리 상태:** ⚠️ 을목 12달 있음. 9개 일간 × 12달 = 108칸 추가 필요.

### 6.2 학파 우선순위 (전통 명리학)

```
1. 변격(종강·종재·화격 등) 후보? 
   → YES: 변격 흐름에 종속 (다른 학파 무시)
   → NO: 다음
2. 조후가 다급? (예: 한겨울 병화, 한여름 임수, 마른 미월 을목)
   → YES: 조후용신 채택
   → NO: 다음
3. 억부용신 (가장 흔한 70%)
4. 격국용신 (정통파 보조)
```

## 7. 십성 조합 패턴

적천수 + 자평진전 표준 패턴들. 사주 8글자 안에 라벨이 트리거.

### 7.1 트리거 거리 — 우리 결정

명리학 책에 거리 명시 없음. 두 가지 학설:
- **거리 무관:** 사주 8글자 안에 X와 Y가 같이 존재하면 트리거 (다수 의견, 보수적)
- **인접 필요:** X와 Y가 옆 자리에 있어야 트리거 (소수 의견, 엄격)

**우리는 거리 무관 채택.** 풀이가 풍성해지고, 명리학 다수 의견.

### 7.2 길격 패턴

## 8. 합·충·형·파·해

### 8.1 합 (合)

**천간합 5쌍:** 갑기→토 / 을경→금 / 병신→수 / 정임→목 / 무계→화.
**지지 육합 6쌍:** 자축→토 / 인해→목 / 묘술→화 / 진유→금 / 사신→수 / 오미→화토.
**삼합 4그룹:** 인오술→화국 / 사유축→금국 / 신자진→수국 / 해묘미→목국. (반합: 2글자만)

### 8.2 충 (沖)

**지지 6충:** 자오 / 축미 / 인신 / 묘유 / 진술 / 사해.

### 8.3 형 (刑)

## 11. 대운(大運)

### 11.1 시작 나이 계산

```
1. 양남(년간 양 + 남자) / 음녀 → 순행
   음남 / 양녀 → 역행
2. 출생일 → 다음 절기(순행) 또는 이전 절기(역행)까지의 일수
3. 일수 ÷ 3 = 대운수 (시작 나이)
4. 나머지: 1 → 버림, 2 → +1
```

12절(節)만 사용. 12중기는 사용 안 함.

### 11.2 대운 기둥

월주에서 시작:
- 순행: 월주 다음 60갑자 (+1, +2, ...)
- 역행: 월주 이전 60갑자 (-1, -2, ...)

각 대운 10년. 보통 8-10개 사이클로 평생 커버.

### 11.3 길흉 색깔

용신·희신을 생/돕는 대운: 길(파랑)
기신·구신을 생/돕는 대운: 흉(빨강)
한신: 평(무색)

평가는 대운의 천간·지지를 일간 기준 십성으로 변환 → 그 십성이 용신/기신 어느쪽이냐로.

**우리 상태:** ✅ `daewoon.ts calculateDaeUn()` 있음. 길흉 자동 평가 추가 필요.
